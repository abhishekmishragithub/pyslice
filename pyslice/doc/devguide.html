1. Overview

Pyslice is implemented in Python. Right now pyslice will only run on UNIX or Linux machines, but that shouldn't be a problem because most Beowulf clusters are UNIX or Linux.

2. Architecture

The overall process:
  1. Parses pyslice.conf file. 'self.read_config()'
    • All quotes within strings in pyslice.conf are removed. 'self.dequote()'
    • All paths are corrected to match the seperator of host machine. 'self.path_correction()'
  2. Create cartesian of sets for variables specified in pyslice.conf. 'self.cartesian()'
  3. Asks user if number of runs is appropriate. If 'n' then bail.
  4. Forces itself into the background. 'self.daemonize()'
  5. Loops on the sets in the cartesian.
    • Loops across all files in template directory.
      • For across variable names in pyslice.conf for each line in the file.
        • Replace all 'keyword' 'Python statement containing variable name' 'keyword' with the restricted evaluation of the Python statement, replacing variable name with correct number from cartesian set.
      • Write out new line to correct output directory and output file.
    • If number of controlled processes is greater than max_processes, wait. 'self.reapChildren()'
    • Create new child process, recording new pid in activeChildren list.
